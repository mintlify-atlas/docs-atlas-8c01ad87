---
title: "Generate, import, view, and delete Seedkeeper secrets"
description: "How to generate, import, view, and delete passwords, mnemonics, wallet descriptors, 2FA secrets, and raw data stored on your Seedkeeper card."
---

Seedkeeper acts as an encrypted vault on a smart card. You can add secrets by generating them inside the application or importing existing ones, then retrieve and view them at any time by entering your PIN. The secrets list screen is your starting point — it shows every secret stored on the card along with its ID, type, and label.

## Listing secrets

Open **My secrets** from the sidebar to see all secrets currently stored on your card. The table shows three columns:

- **ID** — The numeric identifier assigned to the secret at import time.
- **Type of secret** — The secret type (e.g., Password, Mnemonic, Wallet descriptor).
- **Label** — The human-readable name you gave the secret.

Click any row to open the secret's detail view.

## Secret types

<AccordionGroup>
  <Accordion title="Passwords">
    Passwords can be generated randomly inside the application or imported from an existing source.

    **Generating a password**

    Navigate to **Generate** in the sidebar, then select the password generator. Configure the following options:

    - **Label** (required) — A name to identify this password entry, up to 127 bytes.
    - **Login** (optional) — The username or email associated with this credential, up to 255 bytes.
    - **URL** (optional) — The website or service URL, up to 255 bytes.
    - **Length** — Use the slider to choose a length between 6 and 18 characters. The default is 12. Lengths below 8 are flagged in red; 10 or more is highlighted green.
    - **Character sets** — Choose any combination of lowercase (`abc`), uppercase (`ABC`), digits (`123`), and symbols (`#$&`). All four are selected by default.

    Click **Generate** to produce a random password from the selected character pool, then click **Import to card** to save it.

    **Importing an existing password**

    Navigate to **Import**, select **Password**, and fill in the same fields: label, login, URL, and the password text itself. Click **Import to card** to save.

    <Note>
      The **Label** field is mandatory for all password entries. The application will refuse to import a password without a label.
    </Note>

    **Limits:** Password, login, and URL are each capped at 255 bytes. The combined encoded payload must not exceed 255 bytes on Seedkeeper v1.

    **Viewing a password**

    Click the password row in **My secrets**. The detail screen shows the label, login, URL, and password. Use **Hide/Show** to toggle password visibility, and **QR code** to display the password as a scannable QR code.
  </Accordion>

  <Accordion title="BIP39 mnemonics">
    Mnemonics are stored on the card as a masterseed (the derived 64-byte seed) together with the original entropy, optional passphrase, and optional wallet descriptor. This lets you recover both the mnemonic words and the derived keys from a single card entry.

    **Generating a mnemonic**

    Navigate to **Generate** in the sidebar, then select the mnemonic generator. Configure:

    - **Label** (required) — A name for this mnemonic entry, up to 127 bytes.
    - **Word count** — Choose **12 words** (128-bit entropy) or **24 words** (256-bit entropy).
    - **Passphrase** (optional) — Enable the passphrase checkbox to add a BIP39 passphrase, up to 255 bytes.
    - **Descriptor** (optional) — Enable the descriptor checkbox to attach a wallet output descriptor to this entry.

    Click the radio button for your preferred word count to generate a fresh mnemonic, then click **Save on card** to store it.

    **Importing an existing mnemonic**

    Navigate to **Import**, select **Mnemonic seedphrase**, and enter:

    - **Label** (required)
    - **Mnemonic** — Your 12 or 24 BIP39 words. The application validates the mnemonic using the BIP39 word list before importing.
    - **Passphrase** (optional)
    - **Descriptor** (optional)

    Click **Save on card** to store the entry.

    <Note>
      The application validates the mnemonic with the BIP39 English word list. Invalid mnemonics are rejected before any data is written to the card.
    </Note>

    **Limits:** Passphrase up to 255 bytes; descriptor up to 65,535 bytes. The combined payload must not exceed 255 bytes on Seedkeeper v1.

    **Viewing a mnemonic**

    Click the mnemonic row in **My secrets**. The detail screen shows the mnemonic words, optional passphrase, and optional descriptor. Use **SeedQR** to display the entropy as a compact QR code, or **QR code** to display the mnemonic words as a QR code.
  </Accordion>

  <Accordion title="Wallet descriptors">
    A wallet descriptor is a Bitcoin output descriptor string that encodes the derivation path, key, and script type for a wallet. Storing it on Seedkeeper lets you recover your wallet structure without relying on any software keeping state.

    **Importing a wallet descriptor**

    Navigate to **Import**, select **Wallet descriptor**, and fill in:

    - **Label** (required) — A name for this descriptor, up to 127 bytes.
    - **Wallet descriptor** — Paste the full descriptor string.

    Click **Import to card** to save.

    **Limits:** Descriptor up to 65,535 bytes. On Seedkeeper v1, the encoded payload must not exceed 255 bytes.

    **Viewing a descriptor**

    Click the descriptor row in **My secrets**. The detail screen displays the full descriptor text and offers a QR code export.
  </Accordion>

  <Accordion title="Raw data">
    The Data type lets you store arbitrary plain text on the card — useful for notes, recovery codes, or any text you want to keep secure.

    **Importing raw data**

    Navigate to **Import**, select **Data**, and fill in:

    - **Label** (required) — A name for this entry, up to 127 bytes.
    - **Data** — The text content to store.

    Click **Import to card** to save.

    **Limits:** Data content up to 65,535 bytes. On Seedkeeper v1, the encoded payload must not exceed 255 bytes.

    **Viewing raw data**

    Click the data row in **My secrets**. The detail screen displays the stored text and offers a QR code export.
  </Accordion>

  <Accordion title="Public keys">
    Public keys can be stored on the card for use during the encrypted backup process. When two Seedkeepers pair for backup, each card's authentication public key (authentikey) is stored on the other card to establish a shared encryption key.

    **Importing a public key**

    Public key import is handled automatically by the backup wizard. If you need to import one manually, you can do so via the import flow in the source code — label is required, and the key must be raw bytes up to 255 bytes in length.

    **Viewing a public key**

    Click the public key row in **My secrets**. The detail screen displays the key as a hex string and offers a QR code export.
  </Accordion>
</AccordionGroup>

## Export rights

Each secret is stored with an export rights flag that controls whether the plaintext can be read out of the card:

- **`0x01` — Plaintext export allowed.** The application can decrypt and display the secret after PIN verification. This is the default for all secrets imported through Satochip-Utils.
- **`0x02` — Plaintext export not allowed.** The secret can be transferred encrypted to another card during backup but cannot be displayed in plaintext. Attempting to view such a secret shows the message: `Export failed: export not allowed by SeedKeeper policy.`

<Warning>
  If a secret has export rights set to `0x02`, you will not be able to read its value through Satochip-Utils. The secret can still be included in an encrypted backup to another Seedkeeper card.
</Warning>

## Deleting a secret

Open the secret from **My secrets** and click **Delete secret**. A confirmation dialog warns that the operation is irreversible before proceeding.

<Warning>
  Secret deletion is only available on **Seedkeeper v2**. On Seedkeeper v1, the delete button is present in the UI but the operation will fail with an error: `Secret deletion is not supported on Seedkeeper v0.1!`
</Warning>

## Audit logs

Open **Logs** from the sidebar to view the card's operation history. The log table shows the most recent operations recorded by the card's internal tamper-evident log, with four columns:

| Column | Description |
|---|---|
| **Operation** | The operation name (e.g., Import plain secret, Export plain secret, Verify PIN, Reset secret) |
| **ID1** | The secret ID that was the subject of the operation |
| **ID2** | A second secret ID involved in the operation, if any (e.g., the pairing key used during encrypted export) |
| **Result** | The outcome (e.g., OK, PIN failed – N tries remaining, Export not allowed) |

The card stores a fixed number of log entries. When the log is full, the oldest entries are overwritten.
