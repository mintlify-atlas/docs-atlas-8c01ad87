Seedkeeper stores secrets as typed binary records. Each secret has a type code that determines how Satochip-Utils encodes, decodes, and displays the data, and an export rights flag that controls whether the secret can be exported in plaintext. This page documents every type code defined in the Satochip-Utils codebase, along with the payload format and size constraints relevant to each.

## Export rights flags

Every secret stored on a Seedkeeper carries an export rights byte alongside its type code.

| Value | Meaning |
|---|---|
| `0x01` | Plaintext export allowed — the secret can be exported and read by Satochip-Utils |
| `0x02` | Export not allowed — the secret can only be used on-card; export is blocked |

Satochip-Utils sets `0x01` (plaintext export allowed) by default for all secrets it imports.

## Seedkeeper v1 vs v2 payload limit

Seedkeeper v1 (protocol version `< 2`) caps the total encoded payload at **255 bytes** per secret. Seedkeeper v2 (protocol version `>= 2`) removes this cap, allowing large payloads such as long wallet descriptors. If you attempt to import a secret that exceeds 255 bytes on a v1 card, Satochip-Utils raises an error before sending data to the card.

<Note>
  Secret deletion is only supported on Seedkeeper v2. On v1 cards, secrets cannot be removed once written.
</Note>

## Secret types

<AccordionGroup>
  <Accordion title="0x10 — Masterseed">
    A raw BIP32 master seed in binary form. When subtype `0x00` is set, the payload uses a 1-byte size prefix followed by the seed bytes. When subtype `0x01` is set (BIP39 variant), the payload additionally includes the wordlist selector byte, the BIP39 entropy with a 1-byte size prefix, the optional passphrase with a 1-byte size prefix, and an optional wallet descriptor with a 2-byte size prefix.

    **Payload format (subtype 0x00):**
    ```
    [seed_size (1 byte)] [seed_bytes]
    ```

    **Payload format (subtype 0x01 — BIP39):**
    ```
    [seed_size (1 byte)] [seed_bytes]
    [wordlist_selector (1 byte)]
    [entropy_size (1 byte)] [entropy_bytes]
    [passphrase_size (1 byte)] [passphrase_bytes]
    [descriptor_size (2 bytes)] [descriptor_bytes]
    ```
  </Accordion>

  <Accordion title="0x30 — BIP39 mnemonic">
    A BIP39 mnemonic phrase (12 or 24 words) stored as a UTF-8 string, with an optional BIP39 passphrase.

    **Payload format:**
    ```
    [mnemonic_size (1 byte)] [mnemonic_utf8_bytes]
    [passphrase_size (1 byte)] [passphrase_utf8_bytes]
    ```

    **Size limits:**
    - Mnemonic: up to 255 bytes UTF-8 (a 24-word English mnemonic is well under this)
    - Passphrase: up to 255 bytes UTF-8
    - Total payload must not exceed 255 bytes on Seedkeeper v1
  </Accordion>

  <Accordion title="0x40 — Electrum mnemonic">
    An Electrum-format mnemonic phrase stored as a UTF-8 string, with an optional passphrase. Uses the same payload format as BIP39 mnemonic (`0x30`).

    **Payload format:**
    ```
    [mnemonic_size (1 byte)] [mnemonic_utf8_bytes]
    [passphrase_size (1 byte)] [passphrase_utf8_bytes]
    ```
  </Accordion>

  <Accordion title="0x50 — Shamir Secret Share">
    A Shamir Secret Sharing share. Satochip-Utils defines this type code in the type dictionary but does not expose a dedicated import or export UI for it in the current version.
  </Accordion>

  <Accordion title="0x60 — Private Key">
    A raw private key. Defined in the type dictionary. No dedicated import UI is provided in the current version of Satochip-Utils.
  </Accordion>

  <Accordion title="0x70 — Public Key">
    A public key stored in uncompressed binary format with a 1-byte size prefix.

    **Payload format:**
    ```
    [pubkey_size (1 byte)] [pubkey_bytes]
    ```

    **Size limit:** Up to 255 bytes.
  </Accordion>

  <Accordion title="0x71 — Authenticated Public Key">
    A public key with an authentication proof. Defined in the type dictionary. No dedicated import UI is provided in the current version of Satochip-Utils.
  </Accordion>

  <Accordion title="0x80 — Symmetric Key">
    A symmetric encryption key. Defined in the type dictionary. No dedicated import UI is provided in the current version of Satochip-Utils.
  </Accordion>

  <Accordion title="0x90 — Password">
    A password record that bundles a password, an optional login name, and an optional URL into a single secret.

    **Payload format:**
    ```
    [password_size (1 byte)] [password_utf8_bytes]
    [login_size (1 byte)]    [login_utf8_bytes]
    [url_size (1 byte)]      [url_utf8_bytes]
    ```

    **Size limits per field:**
    - Password: up to 255 bytes UTF-8
    - Login: up to 255 bytes UTF-8
    - URL: up to 255 bytes UTF-8
    - Total combined payload must not exceed 255 bytes on Seedkeeper v1
  </Accordion>

  <Accordion title="0x91 — Master Password">
    A master password record. Defined in the type dictionary. No dedicated import UI is provided in the current version of Satochip-Utils.
  </Accordion>

  <Accordion title="0xA0 — Certificate">
    A certificate. Defined in the type dictionary. No dedicated import UI is provided in the current version of Satochip-Utils.
  </Accordion>

  <Accordion title="0xB0 — 2FA secret">
    A two-factor authentication secret stored with a 1-byte size prefix.

    **Payload format:**
    ```
    [secret_size (1 byte)] [secret_bytes]
    ```
  </Accordion>

  <Accordion title="0xC0 — Data">
    Arbitrary UTF-8 text data. The payload uses a 2-byte big-endian size header, allowing data larger than 255 bytes on Seedkeeper v2.

    **Payload format:**
    ```
    [data_size (2 bytes, big-endian)] [data_utf8_bytes]
    ```

    **Size limit:** On Seedkeeper v1, the entire payload (2-byte header + data) must not exceed 255 bytes, limiting the data field to 253 bytes. On Seedkeeper v2, there is no enforced cap.
  </Accordion>

  <Accordion title="0xC1 — Wallet descriptor">
    A Bitcoin wallet output descriptor stored as UTF-8 text with a 2-byte big-endian size header.

    **Payload format:**
    ```
    [descriptor_size (2 bytes, big-endian)] [descriptor_utf8_bytes]
    ```

    **Size limit:** Up to 65535 bytes. On Seedkeeper v1, the entire payload must not exceed 255 bytes, limiting useful descriptor length to 253 bytes. On Seedkeeper v2, descriptors up to 65535 bytes are accepted.
  </Accordion>
</AccordionGroup>
